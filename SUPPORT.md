# Support

This document explains how to get support for **YLCA Agent**.

---

## How to Get Help

All support requests should be submitted via **GitHub Issues** in this repository.

Before opening a new issue:
1. Review the documentation in the `docs/` directory
2. Check existing issues
3. Run the recommended diagnostics

---

## Bug Reports

Bug reports must include:

- Operating system
- Execution mode (local / cloud / hybrid)
- Agent version (if known)
- Model name (if applicable)
- Diagnostic output
- Clear steps to reproduce

Issues missing required information may be closed.

---

## Feature Requests

Feature requests should focus on:

- User-facing improvements
- Stability and reliability
- Diagnostics and observability

Requests for internal logic, prompts, or source code will not be accepted.

---

## What Is Not Supported

- Requests for source code
- Requests for internal prompts or planning logic
- General AI usage questions unrelated to YLCA Agent
- Model training or fine-tuning support

---

## Response Expectations

Support is provided on a best-effort basis.
Response times may vary.
